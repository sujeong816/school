-스마트폰 해킹수법 8가지 적고, 그 중 하나에 관한 아는 것 다 적어라
1. 소셜 엔지니어링(social engineering)
사용자 스스로 문을 열게 하는 것
접근 권한 허용을 사용자가 눌렀을 때 침투

2. 악성 광고(Malvertising) 
알림이나 경고처럼 생긴 광고를 띄워 사용자가 무심코 클릭하게 하거나, 사용자에게 겁을 줘서 클릭하게 만듦

3. 스미싱(Smishing) 
잘못된 링크를 문자 등으로 보내 맬웨어를 장치에 설치하게 함.
사용자의 동의가 없어도 악성 파일을 스마트폰에 설치할 수 있음

4. 맬웨어(Malware) 
기기 제조사의 제한을 해제하는 탈옥이나 루팅을 통해 스마트폰 보안 장벽을 일부러 낮춰 놓은 사람을 타깃으로 함
탈옥이나 루팅은 비공식 출처의 원하는 앱을 설치할 수 있게 하지만, 보안이 취약해진다.

5. 프리텍스팅(Pretexting) 
공격자가 피해자의 개인정보를 짜깁기한 내용으로 사칭하여 통신 서비스 업체 연락, 피해자 계정에 대한 접근 권한을 얻음
통신사가 SIM 교체를 해주면 피해자의 전화, 문자, 인증 메시지 등은 공격자에게 간다

6. 블루투스를 통한 침투
무선 네트워크를 통해 침투. 보통 사람들은 블루투스를 계속 켜놓기 때문에 기기에 접근해서 블루투스로 연락처나 개인정보들을 빼내갈 수 있음

7. 중간자 와이파이 공격
해커들이 공공 와이파이를 통해 스마튼에 침투할 수 있다. 통신망을 통해서라면 스마트폰에 직접 통하지 않더라도 공격 가능
혹은 스마트폰을 공격하지 않더라도 공공 와이파이 공유기의 관리자 비번을 해킹해서 공격자가 점유할 수도 있음

8. 악성 케이블
평범한 충전 케이블을 이용해 스마트폰 해킹. 케이블에 해킹 칩을 탑재해 사용자가 케이블을 연결 시 해킹 시작
특정 프로그램 실행, 사이트 접속, 키 입력값 보기 등 다 가능

-DNS
인터넷에서 도메인 이름을 해당 도메인의 IP 주소로 변환. 사용자는 자동으로 DNS 서버에 연결되고 ISP나 공개 DNS 서비스를 통해 DNS 서버에 접근

1. 도메인 이름 해석 : 사용자가 웹 브라우저에 도메인 입력 시 DNS는 도메인을 IP 주소로 변환하여 사용자가 웹 서버 접속이 가능하게 함
2. 분산된 시스템 : 사용자가 도메인 이름 요청 시 해당 도메인의 IP 주소를 가지고 있는 DNS 서버로부터 응답 받아 사용자에게 전달
3. 캐싱 : 응답 받은 도메인 이름과 IP 주소를 일정 시간동안 캐싱. 동일 도메인을 다음에 요청 시 캐시된 IP 주소를 이용하여 더 빠르게 응답 가능 -> 네트워크 트래픽 감소 및 응답 시간 단축


-메타버스
긍정: 가상 세계에서 무한한 가능성 제공. 실제로는 경험할 수 없는 일 가능
어디서든 자유롭게 소통, 협업 가능
부정: 기술적 문제, 법적 문제. 과도한 시간 소비는 실생활에 방해. 사이버 범죄 위험 증가

문제점: 사이버 보안 문제 - 피싱, 해킹, 신원 도용 등 위험.
기기 접속 문제 - 고사양 하드웨어와 빠른 인터넷 필요. 기술적인 장벽 형성하여 모든 사용자가 동일한 경험 불가능
식별 정보 유출: 사용자 개인정보 유출 시 신원 도용 등 문제 발생
법적 문제: 메타버스는 실세계의 법적 구조를 그대로 적용하기 어렵기 때문에 새로운 규제와 법률이 필요함
